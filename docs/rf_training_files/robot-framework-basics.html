Installing Robot Framework

Now that you have Python, pip, and a virtual environment ready, installing Robot Framework is quick. In this chapter, you'll install Robot Framework into your project, verify it works, and run your very first test from the command line.

Robot Framework is an open-source, generic automation framework built on Python. It's widely used for test automation and robotic process automation (RPA). Its readable syntax, rich ecosystem of libraries, and active community at robotframework.org make it a great choice for both beginners and experienced QA professionals.

By the end of this chapter, you will have a working Robot Framework installation and will have executed your first test from the CLI.

Install Robot Framework

Make sure your virtual environment is activated before installing. You should see (.venv) (or similar) at the beginning of your terminal prompt.

First, make sure pip itself is up to date:

$ pip install --upgrade pip

Then install Robot Framework:

$ pip install robotframework

This installs the latest stable version of Robot Framework into your virtual environment. If you want to pin a specific version (recommended for team projects), run:

$ pip install robotframework==7.4.2
Using uv instead? From your project folder, simply run uv add robotframework. This installs Robot Framework into the project's .venv, records it in pyproject.toml, and locks the exact version in uv.lock — no manual activation needed.
Verify the installation

To confirm Robot Framework is installed correctly, ask it for its version:

$ robot --version

You should see output similar to:

Robot Framework 7.4.2 (Python 3.13.0 on win32)

If you see the version number, you're ready to go.

Using uv? Run uv run robot --version instead. The uv run prefix ensures the command uses your project's isolated environment.
Freeze your dependencies (pip only)

To make your project reproducible for teammates or CI, record your installed packages in a requirements.txt file:

$ pip freeze > requirements.txt

Anyone cloning your project can then recreate the same environment with:

$ pip install -r requirements.txt
Using uv? You can skip this step entirely. uv already tracks your dependencies in pyproject.toml and uv.lock. A teammate only needs to run uv sync.
Create your first test

Inside your project folder, create a new file called first_test.robot with the following content:

*** Settings ***
Documentation    My very first Robot Framework test

*** Test Cases ***
Say Hello
    Log To Console    Hello, Robot Framework!
    Should Be Equal    ${1 + 1}    ${2}

A few things to notice:

  • *** Settings *** and *** Test Cases *** are section headers — they tell Robot Framework what kind of content follows.
  • Say Hello is the name of the test case.
  • Log To Console and Should Be Equal are keywords from Robot Framework's built-in libraries.
  • Arguments are separated from keywords (and from each other) by at least two spaces — this is important!
Run your first test from the CLI

With your virtual environment activated, run:

$ robot first_test.robot

You should see output similar to:

==============================================================================
First Test
==============================================================================
Say Hello                                                             Hello, Robot Framework!
| PASS |
------------------------------------------------------------------------------
First Test                                                            | PASS |
1 test, 1 passed, 0 failed
==============================================================================
Output:  /your/project/path/output.xml
Log:     /your/project/path/log.html
Report:  /your/project/path/report.html

🎉 Congratulations — you've just run your first Robot Framework test!

Using uv? Run uv run robot first_test.robot instead.
Explore the logs and reports

After each test run, Robot Framework generates three files in your project folder:

  • output.xml — machine-readable raw output (used to regenerate reports or merge results)
  • log.html — detailed log with every keyword, argument, and result
  • report.html — high-level summary of test statuses

Open log.html and report.html in your browser to explore them. These files are your best friends when debugging failing tests.

Install additional libraries (optional)

Robot Framework on its own comes with several built-in libraries (like BuiltIn, Collections, String, OperatingSystem). For more advanced use cases, you can add external libraries the same way you installed Robot Framework itself:

$ pip install robotframework-browser
$ pip install robotframework-requests
  • Browser — modern web automation based on Playwright
  • Requests — simple keywords for testing REST APIs

To list all installed Robot Framework libraries, run:

$ pip list | grep robotframework

On Windows, use pip list | findstr robotframework instead.

Using uv? Replace pip install with uv add (e.g. uv add robotframework-browser).

These libraries will be covered in more depth in later chapters and in the dedicated web automation course.

Set up your IDE

A good IDE makes writing Robot Framework tests dramatically faster and more enjoyable. It highlights syntax, autocompletes keywords, navigates to keyword definitions, shows inline documentation, and lets you run and debug tests with a single click.

For Robot Framework, the recommended setup today is Visual Studio Code (VS Code) with the RobotCode extension. RobotCode is an actively maintained, all-in-one extension that combines a language server, debugger, test explorer integration, and a code formatter — everything you need in a single install.

By the end of this chapter, you will have VS Code configured with RobotCode, your Python interpreter connected, and your first test runnable directly from the editor.

Why RobotCode?

RobotCode is the successor to older extensions (such as the Robot Framework Language Server and RED). It is built specifically for modern Robot Framework versions and offers:

  • Syntax highlighting for .robot and .resource files
  • Autocompletion for keywords, variables, and imports
  • Go to definition and Find all references for keywords and variables
  • Inline documentation on hover
  • Test Explorer integration — run and debug tests from VS Code's Testing panel
  • Full debugger with breakpoints, step-through, and variable inspection
  • Built-in code formatter using Robotidy
  • Refactoring tools (rename keyword, rename variable, etc.)
Install Visual Studio Code

If you don't already have VS Code installed:

  1. Go to https://code.visualstudio.com
  2. Download the installer for your operating system
  3. Run the installer and follow the default options

On macOS you can also install it via Homebrew:

$ brew install --cask visual-studio-code

On Linux (Debian/Ubuntu):

$ sudo apt install code
Install the RobotCode extension
  1. Open VS Code
  2. Open the Extensions panel (shortcut: Ctrl+Shift+X on Windows/Linux, Cmd+Shift+X on macOS)
  3. Search for RobotCode
  4. Click Install on the extension by d-biehl
RobotCode automatically installs the official Python extension from Microsoft as a dependency. You don't need to install it separately.
Open your project folder

RobotCode works best when VS Code is opened on your project folder (not a single file).

  • Use File → Open Folder… and select your my-rf-tests folder (the one you created earlier)
  • VS Code will remember this as your workspace

Opening the folder is important because RobotCode reads your Python environment, your .robot files, and any resource/library files from the project root.

Select the correct Python interpreter

RobotCode needs to know which Python to use — specifically, the one inside your virtual environment where Robot Framework is installed.

  1. Open the Command Palette (Ctrl+Shift+P / Cmd+Shift+P)
  2. Type and select Python: Select Interpreter
  3. Choose the interpreter inside your project's .venv folder. It should look like:
    • Windows: .\.venv\Scripts\python.exe
    • macOS/Linux: ./.venv/bin/python

If VS Code doesn't list it automatically, click Enter interpreter path… and point it to the file manually.

Using uv? The .venv folder created by uv lives inside your project and works exactly the same way — just pick it from the list.
Verify syntax highlighting and autocompletion

Open your first_test.robot file. You should immediately see:

  • Coloured keywords, sections, and variables (syntax highlighting)
  • Autocompletion suggestions when you start typing a keyword name
  • Hover tooltips showing keyword documentation when you hover over Log To Console or Should Be Equal

If you don't see any of these, double-check that:

  • The file extension is .robot
  • The bottom-right of VS Code shows Robot Framework as the detected language
  • The correct Python interpreter is selected (see previous step)
Run tests from the Test Explorer

RobotCode integrates with VS Code's built-in Testing panel.

  1. Click the flask icon in the left sidebar (the Testing panel)
  2. You will see your test suites and test cases listed
  3. Click the ▶ Run button next to any test to execute it
  4. Click the 🐞 Debug button to run it with the debugger attached

Test results appear inline next to each test (✔ passed, ✘ failed) and full output is shown in the Test Results tab.

Debug a test with breakpoints

RobotCode supports full debugging of Robot Framework tests:

  1. Open a .robot file
  2. Click in the gutter to the left of any line number to set a breakpoint (a red dot appears)
  3. Open the Testing panel and click the 🐞 Debug button next to the test
  4. When execution hits the breakpoint, you can:
    • Inspect variables in the Variables panel
    • Step over, step into, or continue using the debug toolbar
    • Evaluate expressions in the Debug Console
Format your code with Robotidy

RobotCode includes a built-in formatter powered by Robotidy, which enforces consistent Robot Framework style.

  • Format the current file: Shift+Alt+F (Windows/Linux) or Shift+Option+F (macOS)
  • Format on save: add this to your VS Code settings.json:
{
    "[robotframework]": {
        "editor.defaultFormatter": "d-biehl.robotcode",
        "editor.formatOnSave": true
    }
}
